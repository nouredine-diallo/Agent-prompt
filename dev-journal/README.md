# Journal de développement itératif

Ce dossier contient les notes de développement internes générées pendant la construction et le brainstorming du projet. 
Ils ne font pas partie de la documentation officielle de l'API.

Leur présence illustre le processus de développement et les choix architecturaux intermédiaires, utilisés comme support pour concevoir la logique métier (notamment le système de validation JSON et le pipeline d'ingestion).
